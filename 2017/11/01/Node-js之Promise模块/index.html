Node.js之Promise模块

​ 所谓Promise，简单说就是一个容器，里面保存着某个未来才会结束的事件（通常是一个异步操作）的结果。从语法上说，Promise 是一个对象，从它可以获取异步操作的消息。Promise 提供统一的 API，各种异步操作都可以用同样的方法进行处理。

Promise主要有以下四种用法

  1. 使用then实现链式调用：依次完成串行的任务，前一个任务作为下一个任务的条件和基础
  2. 使用Promise.all实现并行调用，子任务全部完成视为完成，这些子任务之间没有依赖关系
  3. 使用Promise.race实现竞赛调用，只要有一个完成便视为完成，例如在ping server的场景下，选择一个最快的服务器
  4. 使用catch捕捉非正常情况，包括异常和返回非期许的结果

Promise也是有缺点的

  • 无法取消Promise，一旦新建它就会立即执行，无法中途取消。
  • 如果不设置回调函数， Promise内部抛出的错误，不会反应到外部。
  • 当处于Pending状态时， 无法得知目前进展到哪一个阶段（刚刚开始还是即将完成）。
